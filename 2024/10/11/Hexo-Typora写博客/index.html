1.Hexo创建文章、Hexo创建页面

  • 创建文章命令：npx hexo new post 文章名字

​ 根据路径查找，可以看到同时创建了同名的.md文件和文件夹，.md文件用来编写文章，同名的文件夹用来存放文章所涉及到的图片。

  • 创建页面命令：npx hexo new page 页面名称

​ 根据路径查找，可以看到创建了一个search文件夹。

2.编写文章

​ 编写.md文件可以使用Typora编辑器，基于markdown语言进行编写。需要注意的是，如果需要进行分类，需要在文章前面的type处，填写上类名（可以已存在，也可以不存在，相当于新建一个类）。

​ 编写过程中，可能会遇到插入图片失败的问题，解决办法如下：

  • 首先就是，将 Hexo的配置文件_config.yml中的post_asset_folder设置为true。这样每次创建文章的时候，会生成一个同名文件夹来存放图片，以便于图像显示。

  • 下载 hexo-asset-img插件 npm install hexo-asset-imag --save

    hexo-asset-imge插件已被弃用，因此下载hexo-asset-img插件。但我下载完毕后，发现在 node_modules中，两个插件同时存在😥😔

  • 由于当把图片插入到Typora时，一般为绝对地址，且Windows中的地址分隔符为 \ 而Hexo所支持的分隔符为/。因此，需要修改Typora的默认配置。
